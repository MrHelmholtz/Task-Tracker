Определения:
	Goal (цель) - крупное/масштабное/значительное/сложное дело/достижение/задача, требующее для своего достижения, выполнения ряда более простых задач (tasks).
	Task (задача) - дело/этап/задача, которое необходимо выполнить для продвижения в выполнении цели. 
	Library (библиотека) - хранилище всех секций, являющееся базой данных(ядром с информацией) для всего приложения.
	Section (секция) - хранилище всех целей, способствующих развитию в конкретной области.
	Основные сущности - библиотеки, секции, цели и задачи
	Manager (менеджер) - класс для работы с основными сущностями, может добавлять, удалять, изменять библиотеки, секции, цели и задачи. Цели и задачи можно дополнительно завершать.
	Builder (строитель) - класс для создания основных сущностей.
	Printer(принтер) - класс для вывода содержимого основных сущностей.
	Loader (загрузчик) - класс для сохранения и загрузки данных приложения.
	Helper(помошник) - класс для реализации методов и свойств, которые нужны и будут использоваться сразу несколькими другими классами.

Описание классов и интерфейсов:
1. **AbstractAim** - абстрактный класс, задающий базовую структуру всех целей(goals) и задач(tasks), содержит *свойства*:
        - name - имя цели/задачи
        - creationDate - дата создания цели/задачи
        - finishingDate - дата выполнения цели/задачи
        - isCompleted - завершена ли цель/задача
        - comment - комментарий для цели/задачи
		**Наследники**:
		1. **Goal** - класс, описывающий общую структуру целей. Добавляет *свойства*:
			- tasksTotalCount - количество входящих задач
			- completedTasksCount - количество выполненных задач  
			- incompletedTasksCount - количество невыполненных задач
			- tasks - коллекция задач, включенных в эту цель
			Наследники:
			**StrictGoal** - класс, описывающий структуру целей, имеющих крайний срок исполнения. Добавляет *свойства*:
			- deadlineDate - дедлайн, крайний срок исполнения
		2. **Task** - класс, описывающий общую структуру задач. *Не добавляет новых свойств*.
			**Наследники**:
			**StrictTask** - класс, описывающий структуру задач, имеющих крайний срок исполнения. Добавляет *свойства*:
			- deadlineDate - дедлайн, крайний срок исполнения
2. **AbstractVault** - абстрактный класс, описывающий общую структуру для всех хранилищ данных. Содержит свойства:
		 - name - имя хранилища
		 - description - описание, поясняющее для чего хранилище
		 - data - коллекция, хранимых данных
		 Наследники:
		1. **Library** - класс, описывающий структуру библиотек. *Свойства*:
			- data (==override==) - коллекция секций, хранящихся в библиотеке
		2. **Section** - класс, описывающий структуру секций. *Свойства*:
			- data (==override==) - коллекция целей, хранящихся в секции
3. **Manager** - интерфейс, задающий общую структуру поведения менеджеров (managers). Содержит *методы*:
		- add(?) - используется для добавления новой сущности
		- delete(?) - используется для удаления сущности
		- adjust(?) - используется для изменения сущности
		- complete(?) - используется для заверешения сущности
	**Наследники**:
		1. **AppManager** - класс, реализующий основные методы для взаимодействия с основными сущностями.
4. **Builder** - интерфейс, задающий общую структуру поведения строителей (builders). Содержит *методы*:
		- buildLibrary(?) - используется для создания новой библиотеки
		- buildSection(?) - используется для создания новой секции
		- buildGoal(?) - используется для создания новой цели
		- buildTask(?) - используется для создания новой задачи
	**Наследники**:
		1. **AppBuilder** - класс, реализующий основные методы для создания основных сущностей . Содержит ***методы***(помимо переопределенных):
			- getNameAndDescription(String target) - используется для чтения имени и описания(опционально) для объекта класса target. Используется при создании всех сущностей.
5. **Printer** - интерфейс, задающий общую структуру поведения принтеров (printers). Содержит *методы*:
		- print(?) - используется для отображения содержимого сущности.
	**Наследники**:
		1. **AppPrinter** - класс, реализующий основные методы для вывода содержимого основных сущностей .
6. **Loader** - интерфейс, задающий общую структуру поведения загрузчиков (loaders). Содержит *методы*:
		- import(?) - используется для загрузки данных из вне в приложения.
		- export(?) - используется для сохранения (выгрузки данных) из приложения.
		
	**Наследники**:
		1. AppLoader - класс, реализующий основные методы для сохранения и загрузки  основных сущностей .
1. **Helper** - класс, реализующий методы и свойства, необходимые для работы сразу нескольких других классов. Содержит ***поля***:
		- scan - объект класса **Scanner**, использующийся для чтения ввода пользователя
	***Методы***:
		- yesOrNoMenu(String question) - используется для для вывода в консоль меню "Да/Нет" для вопроса, переданного в question.
